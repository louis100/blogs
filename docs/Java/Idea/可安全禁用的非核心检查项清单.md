
以下是 IntelliJ IDEA 中可安全禁用的非核心检查项清单，通过关闭这些功能可显著降低资源消耗：

---

### **推荐关闭的非核心检查项**
1. **未使用声明（Unused declaration）**  
   - 检测未被调用的类/方法/变量  
   - **禁用理由**：频繁扫描整个项目消耗 CPU 资源，可通过手动清理替代  
   - 路径：`Inspections > Java > Declaration redundancy`

2. **拼写检查（Spelling）**  
   - 校验代码注释和字符串的拼写错误  
   - **禁用理由**：实时词典匹配占用内存，对非文档型代码非必需  
   - 路径：`Inspections > Spelling`

3. **重复代码检测（Duplicated code）**  
   - 标识重复代码块  
   - **禁用理由**：跨文件对比计算密集型，大型项目消耗显著  
   - 路径：`Inspections > Java > Data flow analysis`

4. **Java 代码风格检查（Java style guidelines）**  
   - 如缩进、大括号位置等格式规范  
   - **禁用理由**：可通过保存时格式化或团队规范工具替代  
   - 路径：`Inspections > Java > Code style issues`

5. **无条件断点提示（Unconditional breakpoints）**  
   - 标记调试中的无条件断点  
   - **禁用理由**：调试时才需关注，日常开发可关闭  
   - 路径：`Inspections > Java > Probable bugs`

---

### **操作指南**
1. 进入设置：`File > Settings > Editor > Inspections`  
2. 在搜索框输入上述检查项名称  
3. 取消勾选对应项（效果即时生效）  
   ```mermaid
   graph LR
   A[打开Inspections设置] --> B[搜索检查项名称]
   B --> C[取消勾选]
   C --> D[Apply保存]
   ```

>  ?? **优化建议**：关闭后内存占用可降低 10%-25%，尤其对 8GB 以下内存设备效果显著。保留核心检查如`Nullability`、`Resource leaks`确保代码安全性。
